# Deployment Guide ‚Äî SalesFlow CRM on Digital Ocean

Deploys the app to a single Ubuntu droplet at **https://crm.sendrato.com**.

## Stack

| Component   | Technology                        |
|-------------|-----------------------------------|
| OS          | Ubuntu 22.04 / 24.04             |
| Database    | PostgreSQL 16 + pgvector (local)  |
| Runtime     | Node.js 20 LTS + pnpm            |
| Process     | systemd                           |
| Reverse proxy | Nginx                           |
| SSL         | Let's Encrypt (certbot)           |

## Prerequisites

1. **A Digital Ocean droplet** ‚Äî Minimum recommended: 2 GB RAM / 1 vCPU ($12/mo). For heavier AI/LLM workloads or many concurrent users, use 4 GB+.

2. **DNS configured** ‚Äî Create an A record pointing `crm.sendrato.com` to the droplet's IP address. Allow time for DNS propagation before running the script.

3. **SSH access** ‚Äî `ssh root@<DROPLET_IP>`

4. **External service credentials** ready (see Environment Variables below).

## Quick Start

```bash
# 1. Copy script to droplet
scp deploy.sh root@<DROPLET_IP>:/root/deploy.sh

# 2. SSH in
ssh root@<DROPLET_IP>

# 3. Edit the configuration section at the top of the script
#    Fill in your OAuth, Forge API, and other credentials
nano /root/deploy.sh

# 4. Run
chmod +x /root/deploy.sh
/root/deploy.sh
```

The script is idempotent for the git clone step ‚Äî re-running it will pull the latest code and redeploy.

## Environment Variables

These must be set in the configuration section of `deploy.sh` before running (or edited in `/opt/salesflow/.env` after deployment):

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | Auto-generated by the script |
| `JWT_SECRET` | Auto-generated by the script (random 32-byte secret) |
| `VITE_APP_ID` | Your OAuth application ID |
| `OAUTH_SERVER_URL` | URL of your OAuth server |
| `OWNER_OPEN_ID` | OpenID of the admin/owner user |
| `VITE_OAUTH_PORTAL_URL` | OAuth login portal URL (used by frontend) |
| `BUILT_IN_FORGE_API_URL` | LLM/Forge API base URL |
| `BUILT_IN_FORGE_API_KEY` | LLM/Forge API key |
| `VITE_FRONTEND_FORGE_API_KEY` | Frontend Forge API key |
| `VITE_FRONTEND_FORGE_API_URL` | Frontend Forge API base URL |

`DATABASE_URL` and `JWT_SECRET` are generated automatically. All others need your actual values.

**Note:** `VITE_*` variables are baked into the frontend at build time. If you change them after the initial build, you must rebuild:

```bash
cd /opt/salesflow
pnpm run build
systemctl restart salesflow
```

## What the Script Does

1. **System update** ‚Äî Installs base packages (curl, git, build-essential, ufw)
2. **Firewall** ‚Äî Enables UFW, allows SSH and Nginx (80/443)
3. **PostgreSQL 16 + pgvector** ‚Äî Installs, creates database `salesflow` with a dedicated user, random password, and enables the `vector` extension for semantic search
4. **Node.js 20 + pnpm** ‚Äî Installs via NodeSource, enables corepack
5. **Application** ‚Äî Clones the repo to `/opt/salesflow`, writes `.env`
6. **Build** ‚Äî `pnpm install`, `pnpm run build`, `pnpm run db:push` (migrations)
7. **systemd** ‚Äî Creates `salesflow.service` that auto-starts and restarts on failure
8. **Nginx** ‚Äî Reverse proxy on port 80 ‚Üí localhost:3000 with streaming support
9. **SSL** ‚Äî Obtains a Let's Encrypt certificate via certbot, configures HTTPS with auto-redirect

## Operations

### Service management

```bash
systemctl start salesflow
systemctl stop salesflow
systemctl restart salesflow
systemctl status salesflow
```

### View logs

```bash
# Follow live logs
journalctl -u salesflow -f

# Last 100 lines
journalctl -u salesflow -n 100

# Since last hour
journalctl -u salesflow --since "1 hour ago"
```

### Deploy updates

```bash
cd /opt/salesflow
git pull origin main
pnpm install --frozen-lockfile
pnpm run build
pnpm run db:push          # apply any new migrations
systemctl restart salesflow
```

### Database access

```bash
# Credentials are in /opt/salesflow/.env
sudo -u salesflow psql salesflow
```

### SSL certificate

Auto-renews via certbot's systemd timer. Verify:

```bash
systemctl list-timers | grep certbot
certbot renew --dry-run
```

### Backup database

```bash
sudo -u postgres pg_dump salesflow > /root/salesflow-backup-$(date +%Y%m%d).sql
```

## Troubleshooting

| Issue | Check |
|-------|-------|
| App won't start | `journalctl -u salesflow -n 50` ‚Äî look for missing env vars or DB connection errors |
| 502 Bad Gateway | Is the app running? `systemctl status salesflow`. Is it on the right port? Check `PORT` in `.env` |
| SSL cert failed | Is DNS pointing to this server? `dig crm.sendrato.com`. Is port 80 open? `ufw status` |
| DB migration fails | Check `DATABASE_URL` in `.env`. Test with `sudo -u salesflow psql salesflow` |
| Build fails | Check Node/pnpm versions: `node -v`, `pnpm -v`. Ensure enough RAM (2 GB minimum) |
| VITE_* changes not visible | These are compile-time. Rebuild: `pnpm run build && systemctl restart salesflow` |
